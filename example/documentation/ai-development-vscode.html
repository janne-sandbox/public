AI DEVELOPMENT / VS Code setup

Install the editor

Use the official VS Code download. Keep VS Code, its extensions, Git, and the project toolchain updated through the organization's approved software channel.

macOS

  1. Download the macOS .dmg, open it, and drag Visual Studio Code to Applications.
  2. Open VS Code. In the Command Palette, run Shell Command: Install 'code' command in PATH.
  3. Restart the terminal and verify with code --version.

Windows (native development)

  1. Download and run the Windows User Setup installer.
  2. Open a new PowerShell window and verify with code --version.
  3. Keep native Windows projects in a Windows directory such as C:\src.

Ubuntu

Download the current Debian package, then install the downloaded file from its directory:

sudo apt install ./code_*.deb code --version

Red Hat Enterprise Linux

Download the current RPM matching the machine architecture, then install it from its directory:

sudo dnf install ./code-*.rpm code --version

For managed fleets or repeatable updates, use Microsoft's documented apt or yum repository instead of manually replacing packages.

Windows with WSL 2

Architecture: install the VS Code desktop application on Windows, not inside the WSL distribution. The WSL extension installs a small server in Linux so terminals, source control, debuggers, and workspace extensions operate against the Linux project.
  1. Open PowerShell as Administrator, run wsl --install, and restart Windows when requested.
  2. Launch the installed distribution once and create its Linux user.
  3. Install VS Code on Windows and install Microsoft's WSL extension.
  4. Keep Linux projects in the WSL filesystem, for example ~/src/example, rather than under /mnt/c.
  5. From the WSL terminal, open the project with code .. Confirm the lower-left remote indicator shows the WSL distribution.
# PowerShell (Administrator) wsl --install # After restart, inside the WSL terminal mkdir -p ~/src cd ~/src code .

Install language extensions in the WSL environment when VS Code offers Install in WSL. User-interface extensions can remain on the Windows side.

Baseline configuration

  • Install Git and set a name and email appropriate for the repository.
  • Enable format-on-save only when the project already defines a formatter.
  • Enable workspace trust and inspect a repository before trusting it.
  • Install only verified extensions from publishers you recognize.
  • Open the repository root so assistants can find its instructions, build files, and tests.
git --version code --version git status
