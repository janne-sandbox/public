AI DEVELOPMENT / Continue for VS Code

One extension, multiple model providers

Continue is an AI coding assistant for VS Code and JetBrains IDEs. Its local YAML configuration can assign different models to chat, editing, applying changes, autocomplete, embeddings, and reranking.

Different from CONTINUE.md: Continue is the VS Code extension described on this page. Continue across chats describes a project handoff file that can be used with any assistant.

See Models and operating modes for the difference between thinking models, configured roles, and Continue's Chat, Plan, and Agent modes.

Install and open the configuration

  1. Open VS Code Extensions and install Continue from Continue.dev. Verify the publisher before installation.
  2. Open the Continue sidebar and complete its sign-in or onboarding flow.
  3. Open the configuration selector above the chat input, hover over Local Config, and choose the gear icon.
  4. Edit and save config.yaml. Continue normally reloads a saved configuration automatically.
EnvironmentDefault local configuration
macOS, Ubuntu, RHEL, WSL~/.continue/config.yaml
Native Windows%USERPROFILE%\.continue\config.yaml

When VS Code is connected to WSL, use the gear icon to confirm which extension host and configuration file are active. The model endpoint must be reachable from that environment.

Prepare Ollama models

Install and start Ollama as described in Local AI with Ollama. Check available memory and disk space before downloading: the Coder V2 and 30B configurations can be demanding.

ollama pull deepseek-coder-v2 ollama pull deepseek-r1:8b ollama pull qwen3-coder:30b ollama list curl http://localhost:11434

The value of each YAML model field must match the tag shown by ollama list. If Ollama runs on a different host, replace localhost with the approved reachable address and secure the connection.

Main Config sample

This version preserves the requested five models, fixes YAML nesting, and keeps the Anthropic key outside the configuration file.

name: Main Config version: 1.0.0 schema: v1 models: - name: DeepSeek Coder V2 (Local) provider: ollama model: deepseek-coder-v2 apiBase: http://localhost:11434 roles: - chat - edit - apply - name: DeepSeek R1 8B (Local) provider: ollama model: deepseek-r1:8b apiBase: http://localhost:11434 roles: - chat - edit - apply - name: Claude Sonnet 4.6 provider: anthropic model: claude-sonnet-4-6 apiKey: ${{ secrets.ANTHROPIC_API_KEY }} roles: - chat - edit - apply defaultCompletionOptions: contextLength: 200000 maxTokens: 64000 capabilities: - tool_use - image_input - name: Claude Opus 4.6 provider: anthropic model: claude-opus-4-6 apiKey: ${{ secrets.ANTHROPIC_API_KEY }} roles: - chat - edit - apply defaultCompletionOptions: contextLength: 200000 maxTokens: 64000 capabilities: - tool_use - image_input - name: Qwen3 Coder Local provider: ollama model: qwen3-coder:30b apiBase: http://localhost:11434 roles: - chat - edit - apply capabilities: - tool_use - image_input defaultCompletionOptions: contextLength: 32768 maxTokens: 8192
Verify declared capabilities. Adding tool_use or image_input tells Continue to enable corresponding behavior; it does not add that capability to the underlying model. Remove a declaration when the exact Ollama model cannot perform it. Agent mode requires working tool support.

Store the Anthropic key safely

Do not use a literal value such as sk-ant-api03-... in config.yaml. Put the real key in a local .env file that is excluded from version control:

EnvironmentRecommended global secret file
macOS, Ubuntu, RHEL, WSL~/.continue/.env
Native Windows%USERPROFILE%\.continue\.env
ANTHROPIC_API_KEY=replace-with-the-real-key

Continue can also resolve a workspace .env or .continue/.env, but those files must be ignored by Git. Restart or reload VS Code after adding or changing IDE secrets.

Understand roles and costs

Role or capabilityMeaning
chatThe model appears for conversations and Agent/Plan workflows.
editThe model can produce targeted editor changes.
applyThe model can apply generated changes to files.
tool_useContinue may offer tools to the model for Agent mode.
image_inputContinue may send supported image context to the model.
  • Local Ollama requests stay local when the configured endpoint is local, but model downloads and extension services may still use the network.
  • Anthropic requests send selected prompts and context to the hosted API and incur API usage charges.
  • Large context and output limits increase latency, memory use, and hosted cost. Lower them when the task does not need them.
  • Start in Chat or Plan mode, inspect context and commands, and review every diff before accepting it.

Troubleshooting

SymptomCheck
Ollama model not foundRun ollama list and make the YAML tag match exactly.
Cannot reach OllamaRun curl http://localhost:11434 from the same environment as Continue.
Anthropic authentication failsCheck the secret name, .env location, key status, and reload VS Code.
Agent mode unsupportedConfirm real tool support; remove incorrect capabilities or select another model.
Configuration does not loadCheck YAML indentation and use Developer: Reload Window.
