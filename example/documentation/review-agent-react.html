Code Review System - React Agent

React Agent Architecture

The React Agent extends ReviewAgent and scans JavaScript, JSX, TypeScript, and TSX files for accessibility, browser security, performance, organization, and requirement coverage.

Key responsibilities

  • Find image elements without alternative text.
  • Check for ARIA signals in larger React codebases.
  • Detect unsanitized dangerouslySetInnerHTML.
  • Flag authentication tokens stored in browser storage.
  • Suggest splitting large components and using lazy loading.
  • Look for component and custom-hook organization.

Class Structure

class ReactAgent extends ReviewAgent { async run(projectRoot, requirementsDir, outputDir, timestamp) _isExcludedReactPath(filePath) _evaluateReactCoverage(reactFiles) _checkAccessibility(reactFiles) _checkSecurity(reactFiles) _checkPerformance(reactFiles) _checkComponentOrganization(reactFiles) _checkFunctionalRequirements(reactFiles) }

Checks

Accessibility

Every scanned file is checked for image elements without alt. Missing text produces a major finding. In projects with more than ten React files, the absence of ARIA attributes also produces a major finding.

Browser security

Unsanitized dangerouslySetInnerHTML produces a critical finding. Storing tokens through localStorage or sessionStorage produces a major finding.

Performance

The first fifteen files are sampled. Multiple components larger than 5 KB trigger a minor suggestion, as does missing lazy loading in projects with more than ten React files.

Organization

Larger projects receive minor suggestions when their paths show no component folder or custom-hook structure.

Requirement coverage

The first five relevant functional requirements are evaluated against the scanned React files. Missing evidence is reported as a minor finding.

Security and Accessibility Examples

Alternative text

<img src="product-thumbnail.png" alt="Blue travel mug" />

Sanitized rich text

const safeHtml = DOMPurify.sanitize(untrustedHtml); return <div dangerouslySetInnerHTML={{ __html: safeHtml }} />;
The agent recognizes sanitization signals such as DOMPurify, sanitizeHtml, or xss in the sampled files.

Scanning Scope and Limitations

  • The scanner matches .js, .jsx, .ts, and .tsx.
  • Paths under review output, documentation, coverage, distribution, and build folders are excluded.
  • The agent uses regex heuristics rather than the React compiler, ESLint, or an AST.
  • Security, performance, and organization checks cannot prove runtime behavior.
