Workflow: Azure Static Web Apps

Trigger and deployment

azure-static-web-apps.yml runs for every push to main and manual dispatch. It checks out source and calls Azure/static-web-apps-deploy@v1 for a plain static site with no Functions API or build-output directory.

Secrets

AZURE_STATIC_WEB_APPS_API_TOKEN must be stored as a repository or environment secret. GITHUB_TOKEN is supplied by GitHub for repository integration. Never copy either value into YAML or logs.

Known configuration issue

The workflow currently declares app_location: documents, while this repository stores the site in documentation/. Unless a deployment environment supplies that directory, the workflow will fail or deploy the wrong content; align the path before relying on this workflow.
