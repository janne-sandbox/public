Code Review System — Instructions

Basic Usage

Review all languages (default)

cd /path/to/example-project ./run-review.sh

Review single language

# TypeScript only LANGUAGE=typescript ./run-review.sh # Java only LANGUAGE=java ./run-review.sh # React only LANGUAGE=react ./run-review.sh # Rust only LANGUAGE=rust ./run-review.sh # C++ only LANGUAGE=cpp ./run-review.sh # C# only LANGUAGE=csharp ./run-review.sh

Control internet access

# Offline (default) LANGUAGE=typescript INTERNET=false ./run-review.sh # Online Deepseek API (requires DEEPSEEK_API_KEY) export DEEPSEEK_API_KEY='your-key' LANGUAGE=cpp INTERNET=true DEEPSEEK=true ./run-review.sh

Control Deepseek semantic analysis

# Default: DEEPSEEK=true LANGUAGE=all ./run-review.sh # Force regex-only mode LANGUAGE=all DEEPSEEK=false ./run-review.sh # Use offline Deepseek when Ollama is available LANGUAGE=all INTERNET=false DEEPSEEK=true ./run-review.sh # Use hosted Deepseek export DEEPSEEK_API_KEY='your-key' LANGUAGE=csharp INTERNET=true DEEPSEEK=true ./run-review.sh

Combined parameters

# C++ with hosted Deepseek export DEEPSEEK_API_KEY='your-key' LANGUAGE=cpp INTERNET=true DEEPSEEK=true ./run-review.sh # C# deterministic checks only LANGUAGE=csharp INTERNET=false DEEPSEEK=false ./run-review.sh

Reading Review Reports

Reports are written to a timestamped run folder under ./review/results/ with naming pattern:

./review/results//review---.md

If two runs start within the same minute, the later folder gets a numeric suffix such as ./review/results/202605261955-1/ so previous results are not overwritten.

Examples:

./review/results/202605261949/ ├── review-typescript-approved-202605261949.md ├── review-java-approved-202605261949.md ├── review-react-approved-202605261949.md ├── review-rust-approved-202605261949.md ├── review-cpp-approved-with-changes-202605261949.md └── review-csharp-approved-with-changes-202605261949.md

A single-language run produces one file in that folder. LANGUAGE=all produces up to six files.

Report sections

  • Header — timestamp, verdict, internet mode, Deepseek status, analysis method
  • Summary — total findings by severity (critical, major, minor)
  • Suggested Changes — code samples with explanations
  • Detailed Findings — each issue with file, line, description, and requirement reference
  • Requirements Coverage — which requirements were evaluated

Example report header

# Code Review Report — TYPESCRIPT **Timestamp:** 2026-05-26T18:24:00.000Z **Internet Access:** Disabled **Deepseek Analysis:** Disabled **Analysis Method:** Regex **Verdict:** `APPROVED WITH CHANGES` ## Summary - Total findings: 2 - Critical issues: 0 - Major issues: 1 - Minor issues: 1

Interpreting Verdicts

APPROVED

No action required

  • No critical or major issues found
  • Code passes all mandatory checks
  • Ready to merge

APPROVED WITH CHANGES

Review suggestions

  • Major issues found OR suggestions provided
  • Review suggested changes and decide whether to implement
  • Can merge if stakeholders agree no changes are needed

DENIED

Block merge

  • Critical issue detected (security, safety, compilation failure)
  • Must resolve all critical issues before resubmitting for review
  • Cannot merge until verdict changes to APPROVED
Decision matrix: Critical findings always block merge. Major findings can be negotiated. Minor findings are advisory.

Common Findings & Fixes

TypeScript: Strict mode not enabled

Issue: tsconfig.json missing "strict": true

Fix:

{ "compilerOptions": { "strict": true, "noUncheckedIndexedAccess": true, "noImplicitAny": true } }

Java: Manual JSON construction

Issue: Using JSONObject instead of Jackson

Fix:

// ✓ Good: Use Jackson with record classes @RestController public class MyController { @PostMapping public ResponseEntity create(@RequestBody @Valid MyDto dto) { return ResponseEntity.ok(dto); } public record MyDto(String id, String name) { } }

React: Missing alt text on images

Issue: <img> tags without alt attribute

Fix:

// ✗ Bad <img src="product-thumbnail.png" /> // ✓ Good <img src="product-thumbnail.png" alt="Product thumbnail" />

Rust: Unwrap instead of ? operator

Issue: Using .unwrap() in error paths

Fix:

// ✗ Bad let data = fs::read_to_string("file.txt").unwrap(); // ✓ Good let data = fs::read_to_string("file.txt")?;

Integration with Git Workflow

Pre-commit hook example

#!/bin/bash # .git/hooks/pre-commit LANGUAGE=all INTERNET=false ./run-review.sh || exit 1 echo "✓ Code review passed."

GitHub Actions CI/CD example

name: Code Review on: [pull_request] jobs: review: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 - name: Run code review run: LANGUAGE=all INTERNET=false DEEPSEEK=false ./run-review.sh - name: Check for DENIED verdicts run: | if find ./review/results -type f -name 'review-*.md' -print0 | xargs -0 grep -q "DENIED"; then echo "Critical review failures detected." exit 1 fi - name: Upload reports uses: actions/upload-artifact@v3 with: name: review-reports path: ./review/results/
CI/CD best practice: Run review on every PR. Report DENIED as failed check. APPROVED WITH CHANGES requires maintainer override. APPROVED auto-passes.

Troubleshooting

Agent not found error

ERROR: TypeScript agent not found at ./review/agents/typescript-agent.js

Solution: Ensure all agent files exist in ./review/agents/:

chmod +x ./run-review.sh ls -la ./review/agents/ # Includes: base-agent.js, typescript-agent.js, java-agent.js, rust-agent.js, # react-agent.js, cpp-agent.js, and csharp-agent.js

No Node.js installed

node: command not found

Solution: Install Node.js 18+ from nodejs.org

node --version # Should show v18+

Review script not executable

bash: ./run-review.sh: Permission denied

Solution:

chmod +x ./run-review.sh LANGUAGE=typescript ./run-review.sh

Requirements files not found

ERROR: No requirements found in ./requirements/

Solution: Ensure requirements exist:

ls -la ./requirements/functional/ ls -la ./requirements/non-functional/
Note: The functional directory is optional. Agents load every Markdown file present in requirements/functional/ and requirements/non-functional/.

Deepseek not being used

**Deepseek Analysis:** Disabled **Analysis Method:** Regex

Offline solution: Make sure Ollama is running on 127.0.0.1:11434, the model is installed, and Deepseek is enabled:

ollama serve ollama pull deepseek-coder:1.3b-base LANGUAGE=all INTERNET=false DEEPSEEK=true ./run-review.sh

Online solution: Configure an API key and select online mode:

export DEEPSEEK_API_KEY='your-key' LANGUAGE=cpp INTERNET=true DEEPSEEK=true ./run-review.sh

Advanced: Extending Reviews

Add a new requirement check

Edit the relevant agent file and add a new check method:

// In ./review/agents/typescript-agent.js checkNoConsoleLog() { const findings = []; this.files.forEach(file => { const content = fs.readFileSync(file, 'utf8'); if (content.includes('console.log')) { findings.push({ severity: 'minor', title: 'console.log found in production code', file, line: 'N/A', description: 'Remove console.log statements before production', requirement: 'REVIEW-001 TypeScript Standards' }); } }); return findings; }

Add a new requirement file

Create a new REVIEW-00X-*.md file in ./requirements/non-functional/:

# REVIEW-006 Custom Standards ## Python Code Quality - **Type hints:** All functions must have type hints - **Docstrings:** All modules must have docstrings - **Tests:** Must have ≥75% test coverage - **Linting:** Code must pass pylint
Agents auto-load requirements: Any new Markdown file in either configured requirements directory is loaded on the next run.

Performance Tuning

Speed up reviews

  • Use INTERNET=false (default) to keep model requests local
  • Review single language instead of all: LANGUAGE=typescript
  • Clear old reports: rm -rf ./review/results/*

Expected times

Single language (offline): 5-15 seconds All languages (offline): 30-60 seconds Hosted Deepseek: depends on API latency and selected model
