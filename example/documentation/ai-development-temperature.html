AI DEVELOPMENT / What temperature means

A sampling control, not intelligence

A language model produces probabilities for possible next tokens. Temperature changes how sharply the runtime samples from that distribution. Lower values favor the most likely choices; higher values allow more lower-probability choices and therefore more variation.

TemperatureTypical effectUseful starting point
LowMore focused and repeatable wording.Code review, extraction, classification, and precise edits.
MediumA balance of consistency and alternatives.Implementation ideas, documentation, and ordinary chat.
HighMore diverse and surprising output, with more drift risk.Brainstorming names, examples, or competing designs.
Temperature does not add knowledge or reasoning. It cannot make a weak prompt correct, recover missing context, or prove generated code. Model choice, instructions, context, tools, tests, and review matter more.

Illustrative starting ranges

TaskStarting rangeWhy
Security or requirements review0.0–0.2Prefer focused, comparable output.
Code generation and test cases0.1–0.4Allow limited alternatives without excessive drift.
Documentation drafts0.3–0.7Permit wording variety while retaining structure.
Open brainstorming0.7–1.0Explore more diverse candidates.

These are experiments, not universal standards. Scales, defaults, and supported ranges differ by model and provider. Compare outputs on a representative evaluation set before changing a production setting.

Where can it be configured?

  • Ollama: exposes a temperature option through a Modelfile and generation API.
  • Hosted model APIs: support is endpoint- and model-specific. Consult the selected model's current schema; omit unsupported parameters.
  • Codex, GitHub Copilot, and Claude Code editor experiences: do not assume a raw temperature control exists. Their model and agent settings are not interchangeable with an API sampling parameter.
Do not invent configuration keys. A setting named temperature in one provider's API or local runtime does not imply that another editor extension accepts it. Reasoning effort, model selection, top-p, and temperature are different controls.

Ollama example

Create a review-focused model definition:

FROM deepseek-coder:1.3b-base PARAMETER temperature 0.2 PARAMETER seed 42 SYSTEM You are a precise code reviewer. Cite the supplied code and requirements.

Save it as Modelfile.review, then create and run it:

ollama create deepseek-review -f Modelfile.review ollama run deepseek-review

A fixed seed may improve repeatability in one runtime, but it does not guarantee identical results across model versions, hardware, context changes, tool calls, or software updates.

Evaluate instead of guessing

  1. Prepare representative prompts with expected checks or acceptance criteria.
  2. Keep the model, prompt, context, seed, and software version fixed.
  3. Run several samples at each candidate temperature.
  4. Score correctness, requirement coverage, false positives, consistency, and useful variety.
  5. Select the lowest-variance setting that still produces the diversity the task needs.
