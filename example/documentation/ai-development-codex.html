AI DEVELOPMENT / VS Code with Codex

Install and sign in

  1. Complete the VS Code setup.
  2. Open Extensions and install Codex – OpenAI's coding agent. Verify that OpenAI is the publisher.
  3. Choose the Codex icon. If it is hidden, use the Command Palette and run Codex: Open Codex Sidebar.
  4. Sign in with ChatGPT, or select API-key authentication when that is the approved account model.
  5. Open a project folder and start with an explanation or a small, reversible edit.

The same extension workflow applies on macOS, native Windows, Ubuntu, and RHEL. For WSL, open a WSL window first with code . and install/enable the extension for that remote environment if VS Code prompts.

Give useful context

Codex can work with open files and selected code. State the desired behavior, constraints, files in scope, and validation command. Point it to repository instruction files before requesting edits.

Read README.md and the repository instructions. Explain how authentication reaches the selected endpoint. Do not edit anything yet. Identify the relevant files, tests, security boundary, and the command that validates this path.

After the explanation is correct, request one change:

Add the missing unauthorized-response test. Keep production behavior and public APIs unchanged. Run only the focused test first, then report changed files and results.

Review and permissions

  • Inspect every proposed diff; do not approve based only on the summary.
  • Use the narrowest filesystem and command permissions that complete the task.
  • Keep network access off unless downloads or current external information are required and approved.
  • Never paste secrets into chat. Use the project's secret-management mechanism.
  • Run tests, linters, and security checks independently before committing.
Authentication choice matters: ChatGPT subscription access and API-key usage can have different billing and administrative controls. Follow the organization's account and cost policy.
