Code Review System - Sample Applications

Purpose

The sample-apps/ directory contains seven small applications used as end-to-end review fixtures. Each project has an intentional finding that proves its matching agent scanned and evaluated real source files.

Educational fixtures: Several samples deliberately contain insecure or discouraged code. They exist to test findings and must not be treated as production examples.

Run Verification

The Node integration suite invokes the appropriate agent directly for each isolated sample and asserts both the finding title and generated Markdown report.

npm test

To scan all samples together through the shell orchestrator:

DEEPSEEK=false INTERNET=false LANGUAGE=all ./run-review.sh

Use one agent for a focused run:

DEEPSEEK=false LANGUAGE=cpp ./run-review.sh DEEPSEEK=false LANGUAGE=csharp ./run-review.sh DEEPSEEK=false LANGUAGE=typescript ./run-review.sh

Application-Level Checks

The samples are deliberately dependency-light. Local verification covers:

  • C++ configure, compile, and execution with CMake.
  • C# restore, compile, and execution with the .NET 8 SDK.
  • Java compilation and execution with javac.
  • Rust compilation and tests with Cargo.
  • Browser JavaScript and React fixture tests with Node's built-in test runner.
  • Angular package and TypeScript configuration parsing; the review integration validates its TypeScript sources without installing Angular packages.
