AI DEVELOPMENT / VS Code with GitHub Copilot

Install and authenticate

  1. Use a current VS Code release and a GitHub account with Copilot access.
  2. Open Extensions and install GitHub Copilot. Verify that GitHub is the publisher.
  3. Choose the Copilot icon in the status bar or title bar and sign in to GitHub.
  4. Confirm the account and organization shown by VS Code are the intended ones.

Use the same extension on macOS, native Windows, Ubuntu, and RHEL. In a WSL window, install Copilot in the location VS Code recommends so it can operate on the Linux workspace.

Example workflow

Plan a fix for the failing parser test. Limit the plan to parser.ts and parser.test.ts. Preserve the public API and list the validation commands. Do not edit files until the plan is accepted.
  1. Review the plan and remove unrelated work.
  2. Switch to Agent mode only when the planned edits are acceptable.
  3. Review changed files in Source Control.
  4. Run the test and static checks yourself.
  5. Commit only a coherent, reviewed change.
Generated completion is not provenance proof. Check licenses, security, correctness, and project conventions; do not assume a suggestion is original or compatible with the repository's license.
