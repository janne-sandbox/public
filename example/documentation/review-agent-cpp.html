Code Review System - C++ Agent

Scope

The C++ agent scans C and C++ source and header files: *.c, *.cc, *.cpp, *.cxx, *.h, *.hh, *.hpp, and *.hxx.

LANGUAGE=cpp ./run-review.sh # Aliases LANGUAGE=c++ ./run-review.sh LANGUAGE=cxx ./run-review.sh

Deterministic Checks

  • Unsafe or unbounded C functions such as gets, strcpy, strcat, sprintf, and scanf.
  • Raw new allocations when no smart-pointer ownership is present.
  • C-style casts that should be replaced by explicit C++ casts.
  • Missing source files whose names indicate tests or specifications.
  • Evidence for the first five language-relevant functional requirements.

These checks run with DEEPSEEK=false and remain active if a configured model is unavailable.

Deepseek Review

When Deepseek is available, the agent uses it to select C++-relevant requirements and semantically reviews bounded samples from up to six files. Findings are accepted only from a validated JSON array and are limited to ten per run.

# Offline: source stays local ollama pull deepseek-coder:1.3b-base LANGUAGE=cpp INTERNET=false DEEPSEEK=true ./run-review.sh # Online: sampled source is sent to the Deepseek API export DEEPSEEK_API_KEY='your-key' LANGUAGE=cpp INTERNET=true DEEPSEEK=true ./run-review.sh
Online data boundary: online mode sends requirement headings and sampled source text to the hosted Deepseek API. Do not enable it for code that policy forbids sending to an external service.

Limitations

  • The agent uses text heuristics rather than Clang, compiler diagnostics, sanitizers, or an AST.
  • A filename-based test check does not measure executed coverage.
  • Deepseek findings are suggestions and require developer review.
