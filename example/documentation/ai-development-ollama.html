AI DEVELOPMENT / Local AI with Ollama

What Ollama does

Ollama downloads and runs supported language models, supplies a command-line interface, and exposes a local HTTP API. It is a model runtime rather than a complete code-review policy: model quality, context size, tool integration, licensing, and hardware requirements vary by model.

This example's review tool can use Ollama for offline Deepseek analysis. With the default configuration, it expects deepseek-coder:1.3b-base at http://localhost:11434.

Plan storage before downloading. Models may require many gigabytes or tens of gigabytes. Confirm the model's license permits the intended use, and do not expose the Ollama API beyond loopback without authentication and network controls.

Install by platform

macOS

  1. Check the current macOS and processor requirements in the official guide.
  2. Download and mount ollama.dmg, then drag Ollama to Applications.
  3. Start the app and allow it to add the ollama CLI to the path when prompted.

Windows (native)

  1. Download and run OllamaSetup.exe. Administrator access is not normally required.
  2. Start Ollama, open a new PowerShell window, and run ollama -v.
  3. The native service publishes its API at http://localhost:11434.

Windows with WSL

Prefer the native Windows Ollama application when you want Windows-managed GPU support. Run the review tool from Windows, or explicitly configure and secure connectivity from WSL. If the tool expects WSL's own localhost, install Ollama inside WSL using the Linux method instead. Do not assume Windows and WSL loopback behavior is identical on every networking mode.

Ubuntu and Red Hat Enterprise Linux

Ollama documents one Linux installer for both distributions:

curl -fsSL https://ollama.com/install.sh | sh ollama -v sudo systemctl status ollama

Review the official manual-install and systemd instructions when installer scripts are not allowed. GPU acceleration needs supported hardware and current vendor drivers; CPU execution is slower but useful for small models.

Download and test the review model

ollama pull deepseek-coder:1.3b-base ollama list ollama run deepseek-coder:1.3b-base

Enter a short prompt, then type /bye. Verify the API independently:

curl http://localhost:11434/api/generate \ -d '{"model":"deepseek-coder:1.3b-base","prompt":"Return only OK","stream":false}'

Then run the example's offline review from example/:

LANGUAGE=cpp DEEPSEEK=true INTERNET=false ./run-review.sh LANGUAGE=csharp DEEPSEEK=true INTERNET=false ./run-review.sh

Inspect the generated report's provider status. If Ollama or the model is unavailable, the review tool deliberately falls back to deterministic checks.
