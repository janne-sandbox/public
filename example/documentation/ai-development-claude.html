AI DEVELOPMENT / VS Code with Claude

Use the VS Code extension

The graphical Claude Code extension is the recommended Anthropic workflow in VS Code. It includes its own CLI for the chat panel; install the standalone CLI only if you also want to run claude in a terminal.

  1. Use VS Code 1.94 or newer and an account supported by Claude Code.
  2. Open Extensions, search for Claude Code, and verify Anthropic as the publisher before installing.
  3. Open a file and choose the Spark icon, then sign in.
  4. Select code or mention a file or folder to provide focused context.
  5. Keep Manual or Plan permission mode for the first tasks and inspect side-by-side diffs.

The extension works across macOS, native Windows, Linux, and WSL. For a WSL project, open the folder through the VS Code WSL extension first.

Optional standalone CLI

Installer scripts execute downloaded code. Use only the official Anthropic URLs, inspect scripts when policy requires it, or use the approved Homebrew, WinGet, apt, or dnf package channel.

macOS

brew install --cask claude-code claude --version claude

The official native installer is also available if Homebrew is not used.

Windows (native PowerShell)

winget install Anthropic.ClaudeCode claude --version claude

Native Windows is suitable for Windows-hosted projects. Git for Windows is optional; without it, Claude Code can use PowerShell for shell operations.

Windows with WSL

Install and launch Claude inside the WSL terminal—not PowerShell—when the project is stored in WSL:

curl -fsSL https://claude.ai/install.sh | bash claude --version claude

Ubuntu and Red Hat Enterprise Linux

The same native installer supports Linux. Anthropic also publishes signed apt and dnf repositories for managed package updates.

curl -fsSL https://claude.ai/install.sh | bash claude --version claude

Plan, edit, and review

Claude Code offers permission modes. Start in Plan when the task spans files or involves architecture; use Manual when you want approval before edits and most shell commands.

Inspect the selected module and its tests. Create a plan to handle the timeout without changing the API. Call out concurrency and cleanup risks. Wait for plan approval before editing.
  • Correct the plan before permitting edits.
  • Review each side-by-side change; accept, reject, or revise it.
  • Keep automatic-edit modes for low-risk, well-tested tasks only.
  • Run the repository checks outside the assistant session before commit.
