Parentheses override precedence. Therefore 2^3^2 is 2^(3^2), while -2^2 is -(2^2).

Significant-digit precision

Equation.calculate(precision) selects output precision up to the configured maximum. The engine calculates with a dynamic scale and 16 guard digits, then rounds HALF_EVEN. Canonical output removes redundant leading and fractional trailing zeroes.

Equation.parse("1/3").calculate(5)  // 0.33333
Equation.parse("1e-2000/2").calculate(20) // significant digit retained

Precision-aware π

Number.pi() uses the configured maximum. A parsed pi node uses the precision of that calculation.

Number.setMaxLength(5000);
Number pi = Number.pi();                    // 5,000 significant digits
Number twicePi = Equation.parse("2*pi").calculate(1200);

Stable errors

Malformed expressions, invalid numbers, division by zero, domain errors, and exceeded precision produce UncalculableException with a stable CalculationError. Parser failures also carry a character position. Primitive conversions either round-trip exactly or throw PrecisionLossException with the nearest representable value.
