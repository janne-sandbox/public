Chapter 02.1 · Architecture

Inside the calculation engine.

Public values stay canonical strings. Evaluation converts them into Math2-owned signed fixed-point values backed by base-1,000,000,000 limbs.

No delegated numeric engine

The evaluator does not call Apfloat, BigDecimal, BigInteger, or BigDecimalMath. A recursive bytecode test inspects the engine and nested implementation classes to prevent those dependencies from returning.

Precision and practical limits

The configured maximum controls significant digits, not magnitude. Dynamic working scale preserves values such as 1e-2000. Sixteen guard digits reduce intermediate rounding error before the requested HALF_EVEN result is produced.

Finite machine, configurable precision. “Arbitrary precision” means the caller selects precision rather than using a fixed primitive width. Available heap, array/string capacity, and execution time remain physical limits.
