Cache identity

The key is SHA-256 over engine version, requested precision, and canonical expression. Equivalent whitespace produces the same expression. Changing engine behavior increments the version and prevents stale results from crossing implementations.

SHA-256(engineVersion + "\n" + precision + "\n" + canonicalExpression)

Stored evidence

Each entry records the canonical expression, precision, engine version, unbounded result text, creation/access timestamps, and calculation duration in milliseconds. Indexes support canonical-expression lookup and least-recently-used ordering.

Optional cache

Set MATH2_CACHE_ENABLED=false to calculate without initializing or using the datasource, Hibernate repositories, or Liquibase. A positive MATH2_CACHE_MAX_ENTRIES bounds persistent entries; excess rows are evicted by oldest access and creation time.
