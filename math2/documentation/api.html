Chapter 04 · REST API

A small HTTP surface with exact string results.

Every success value is JSON text, so the caller’s JSON parser never silently rounds a 1,000-digit result.

REST chapterEndpoint referenceWorked examplesErrorsLimits

Base URL

http://127.0.0.1:8080

Loopback is the default. TLS, authentication, and remote exposure belong at a deliberately configured deployment boundary.

Single equation

GET /api/v1/calculate?equation={expression}&precision={digits}

200 OK
{"result":"14"}

precision is optional and defaults to the configured maximum.

Batch

GET /api/v1/calculate/batch?equations={comma-separated expressions}&precision={digits}

200 OK
{"results":["2","3","4"]}

The splitter preserves commas inside functions, so 1+1,max(2,3),sqrt(16) is three equations. Results preserve request order.

Error response

400 Bad Request
{
  "code": "INVALID_EXPRESSION",
  "message": "...",
  "position": 4
}

Known codes include INVALID_REQUEST, INVALID_NUMBER, INVALID_EXPRESSION, DIVISION_BY_ZERO, DOMAIN_ERROR, and PRECISION_EXCEEDED. Messages are safe for external callers.
