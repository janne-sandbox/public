Chapter 01 · Guide

Choose the shortest path to a result.

Use the Java library directly when calculation lives in the JVM. Run the REST service when several languages need the same engine and cache.

In this chapterRequirementsLocal serviceJava libraryDockerVerification

Requirements

Java 21 or newer and Maven 3.9 or newer are sufficient for the Java modules and service. Client toolchains are only needed for the corresponding language.

Precision model. The default maximum is 1,000 significant digits. Change it in-process with Number.setMaxLength or for the service with MATH2_MAX_LENGTH.

Start the local service

./run-dev-local.sh
curl --get --data-urlencode 'equation=sqrt(9)+2^3' --data-urlencode 'precision=1000' http://127.0.0.1:8080/api/v1/calculate
# {"result":"11"}

./stop-dev-local.sh

The service binds to 127.0.0.1. OpenAPI is available at /v3/api-docs and Swagger UI at /swagger-ui.html.

Calculate inside Java

import com.nologo2.math2.Equation;
import com.nologo2.math2.Number;

Equation equation = Number.fromInteger(9).sqrt()
    .add(Number.fromInteger(2).power(Number.fromInteger(3)));

Number result = equation.calculate(1000);
System.out.println(result); // 11

Alternatively, parse the same syntax accepted by REST:

Number circumference = Equation.parse("2*pi*10").calculate(1000);

Run with Docker

docker build -f java-rest-service/Dockerfile -t math2:local .
docker run --rm -p 127.0.0.1:8080:8080 -v math2-data:/app/data math2:local

Verify the checkout

./test-java.sh               # Java library and REST service
./run-smoke-test.sh         # packaged REST smoke test
./test-client-integration.sh # every native client against one live service
./test-all.sh                # complete Java and cross-language verification
