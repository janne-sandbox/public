Chapter 02 · Architecture

One engine. Two ways in. Six language surfaces.

The Java API owns calculation. The REST service adds validation, persistence, caching, and a stable string-based boundary.

ArchitectureSystem viewCalculation engineREST and cache

System context

flowchart LR
  JVM[Java application] --> API[java-api]
  JS[JavaScript / TypeScript] --> REST
  NET[.NET] --> REST
  CPP[C++] --> REST
  RUST[Rust] --> REST
  API --> ENGINE[Math2 decimal engine]
  REST[java-rest-service] --> API
  REST --> H2[(H2 calculation cache)]

Java callers can avoid networking and evaluate directly. All other clients compose the same equation grammar locally, URL-encode it, and receive canonical decimal strings from REST.

Module ownership

ModuleResponsibilityDoes not own
java-apiNumbers, equations, parser, operations, precision, decimal algorithmsHTTP or persistence
java-rest-serviceValidation, endpoints, cache policy, H2, Liquibase, OpenAPINumeric algorithms
Native clientsImmutable values, expression composition, transport, error mappingServer-side calculation

  S->>D: lookup(key)

    D-->>S: canonical decimal result
  else cache miss
    S->>E: evaluate with requested precision
    E-->>S: Number string
    S->>D: persist result + duration
    S->>D: evict least recently used excess
  end
  S-->>C: JSON string result
