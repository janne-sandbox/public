Chapter 04.1 · REST API

Worked HTTP examples.

Use proper query encoding; operators such as + otherwise have special URL form semantics.

REST chapterEndpoint referenceWorked examplesfetchError handling

curl: high-precision π

curl --get \
  --data-urlencode 'equation=2*pi' \
  --data-urlencode 'precision=1000' \
  http://127.0.0.1:8080/api/v1/calculate

The response contains a result string with 1,001 characters: one decimal point plus 1,000 significant digits.

curl: nested batch

curl --get \
  --data-urlencode 'equations=1+1,max(2,3),sqrt(16)' \
  --data-urlencode 'precision=100' \
  http://127.0.0.1:8080/api/v1/calculate/batch

{"results":["2","3","4"]}

Browser or Node fetch

const url = new URL("http://127.0.0.1:8080/api/v1/calculate");
url.searchParams.set("equation", "sqrt(2)");
url.searchParams.set("precision", "1000");

const response = await fetch(url, { headers: { accept: "application/json" } });
const body = await response.json();
if (!response.ok) throw new Error(`${body.code}: ${body.message}`);
console.log(body.result); // keep this as a string

Structured domain failure

curl --get --data-urlencode 'equation=1/0' \
  http://127.0.0.1:8080/api/v1/calculate

HTTP/1.1 400
{"code":"DIVISION_BY_ZERO","message":"Division by zero"}
Prefer a native client. The JavaScript, TypeScript, .NET, C++, and Rust clients already implement URL encoding, timeouts, response validation, batch cardinality checks, and structured service errors.
