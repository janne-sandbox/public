Collection method

DocumentationPerformanceTest starts the Spring context with Liquibase-managed H2, warms class loading with one discarded calculation, evaluates each expression once through CalculationService, and reads the persisted calculation_cache rows through the repository.

mvn -q -pl java-rest-service -am \
  -Dtest=DocumentationPerformanceTest \
  -Dsurefire.failIfNoSpecifiedTests=false \
  test -Ddebug=false

The sample was collected on 2026-07-21. The service measures only equation evaluation using System.nanoTime(), converts it to whole milliseconds, then stores it in H2. HTTP, JSON, cache lookup, and persistence time are excluded.

Interpretation

  • A stored 0 ms means less than one millisecond because persistence uses whole milliseconds.
  • π and square root execute iterative arbitrary-precision algorithms; work grows with requested digits.
  • A cache hit returns the stored string and does not recompute the expression. The schema retains the original miss duration, not cache-hit latency.
  • JIT state, machine load, heap, expression shape, and precision all affect timings. Use JMH or an external load tool before making capacity decisions.
